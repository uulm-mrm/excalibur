Calibration Manager
===================

The calibration results are stored in *yaml* files with the following format:

.. code-block:: yaml

   parent1:
     child1: [transformation]
     child2: [transformation]
   parent2:
     child3: [transformation]
   child2:
     child4: [transformation]

All entries represent a transformation from a ``parent`` to a ``child`` frame.
Each transformation consists of two keys: a ``data`` key, which contains a list of type-specific values generated by the ``motion3d`` export, and a ``type`` key that provides the corresponding transformation type.
The available types are:

.. list-table::

   * - ``A``
     - :py:class:`motion3d.AxisAngleTransform`
   * - ``D``
     - :py:class:`motion3d.DualQuaternionTransform`
   * - ``E``
     - :py:class:`motion3d.EulerTransform`
   * - ``M``
     - :py:class:`motion3d.MatrixTransform`
   * - ``Q``
     - :py:class:`motion3d.QuaternionTransform`

The scripts in ``scripts/transforms`` can be used to merge or visualize calibration files.
